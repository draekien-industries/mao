    1. Change `asar: true` to an object with `unpack` glob: `asar: { unpack: "*.{node,dylib}" }` -- this extracts native `.node` binaries to `app.asar.unpacked/` so they can be loaded at runtime.
    2. Add an `ignore` function that preserves `better-sqlite3` and its transitive native dependencies in `node_modules` during packaging. The Forge Vite plugin aggressively prunes node_modules (issue #3738), so we must explicitly keep these modules:
    ```typescript
    packagerConfig: {
      asar: {
        unpack: "*.{node,dylib}",
      },
      ignore: (file: string) => {
        if (!file) return false;
        // Allow Vite build output
        if (file.startsWith("/.vite")) return false;
        // Selectively keep native module dependencies
        if (file.startsWith("/node_modules")) {
          const parts = file.split("/");
          const moduleName = parts[2];
          const keepModules = [
            "better-sqlite3",
            "bindings",
            "file-uri-to-path",
            "prebuild-install",
            "node-addon-api",
          ];
          return !keepModules.some((m) => moduleName === m);
        }
        return true;
      },
    },
    ```
    Note: The `ignore` function returns `true` to EXCLUDE a file, `false` to INCLUDE it. The logic above includes the root, `.vite/`, and the specific native modules while excluding everything else from `node_modules/`.

    Do NOT remove the existing `FusesPlugin` or `VitePlugin` from the plugins array. Do NOT add `AutoUnpackNativesPlugin` to the plugins array (it has a known regression #3934 and is not needed with the manual ASAR config).

    **Step 2: Create src/services/database/errors.ts**
    Follow the exact pattern from `src/services/claude-cli/errors.ts`. Define three TaggedError classes, a union schema, the inferred type, and a format function:

    ```typescript
    import { Schema } from "effect";

    export class DatabaseOpenError extends Schema.TaggedError<DatabaseOpenError>()(
      "DatabaseOpenError",
      { message: Schema.String, cause: Schema.String },
    ) {}

    export class DatabaseCorruptionError extends Schema.TaggedError<DatabaseCorruptionError>()(
      "DatabaseCorruptionError",
      { message: Schema.String },
    ) {}

    export class DatabaseQueryError extends Schema.TaggedError<DatabaseQueryError>()(
      "DatabaseQueryError",
      { message: Schema.String, cause: Schema.String },
    ) {}

    export const DatabaseErrorSchema = Schema.Union(
      DatabaseOpenError,
      DatabaseCorruptionError,
      DatabaseQueryError,
    );

    export type DatabaseError = Schema.Schema.Type<typeof DatabaseErrorSchema>;

    export function formatDatabaseError(err: DatabaseError): string {
      switch (err._tag) {
        case "DatabaseOpenError":
          return `Failed to open database: ${err.message}`;
        case "DatabaseCorruptionError":
          return `Database corruption detected: ${err.message}`;
        case "DatabaseQueryError":
          return `Database query failed: ${err.message}`;
      }
    }
    ```

    **Step 3: Create src/services/database/schema.ts**
    Export the CREATE TABLE SQL statements as named string constants. The table schemas are derived from REQUIREMENTS.md:

    - **events table** (EVNT-01 through EVNT-04): `id` (INTEGER PRIMARY KEY AUTOINCREMENT), `session_id` (TEXT NOT NULL -- partitions events per session per EVNT-03), `sequence_number` (INTEGER NOT NULL -- strict ordering per EVNT-04), `event_type` (TEXT NOT NULL -- event type discriminator), `event_data` (TEXT NOT NULL -- full JSON event payload), `created_at` (TEXT NOT NULL DEFAULT datetime('now') -- timestamp). UNIQUE constraint on (session_id, sequence_number) to enforce ordering uniqueness.
    - **tabs table** (TAB-01): `id` (INTEGER PRIMARY KEY AUTOINCREMENT), `session_id` (TEXT -- nullable; a tab may exist before a session starts), `cwd` (TEXT NOT NULL -- repository/working directory), `git_branch` (TEXT -- nullable; not all projects use git), `display_label` (TEXT -- nullable; user-facing label), `tab_order` (INTEGER NOT NULL -- position in tab bar), `is_active` (INTEGER NOT NULL DEFAULT 0 -- boolean flag for active tab), `created_at` (TEXT NOT NULL DEFAULT datetime('now')), `updated_at` (TEXT NOT NULL DEFAULT datetime('now')).

    Index on events.session_id for fast session-scoped queries:

    ```typescript
    export const EVENTS_TABLE_SQL = `
      CREATE TABLE IF NOT EXISTS events (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        session_id TEXT NOT NULL,
        sequence_number INTEGER NOT NULL,
        event_type TEXT NOT NULL,
        event_data TEXT NOT NULL,
        created_at TEXT NOT NULL DEFAULT (datetime('now')),
        UNIQUE(session_id, sequence_number)
      )
    `;

    export const EVENTS_SESSION_INDEX_SQL = `
      CREATE INDEX IF NOT EXISTS idx_events_session_id
      ON events (session_id)
    `;

    export const TABS_TABLE_SQL = `
      CREATE TABLE IF NOT EXISTS tabs (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        session_id TEXT,
        cwd TEXT NOT NULL,
        git_branch TEXT,
        display_label TEXT,
        tab_order INTEGER NOT NULL,
        is_active INTEGER NOT NULL DEFAULT 0,
        created_at TEXT NOT NULL DEFAULT (datetime('now')),
        updated_at TEXT NOT NULL DEFAULT (datetime('now'))
      )
    `;
    ```
