### Phase-Specific Semantic Colors

These are custom status colors for this phase's indicator dots. They do NOT replace existing shadcn tokens.

- **Status green (unread):** `var(--sidebar-primary)` -- Reuses sidebar-primary. The existing `SessionStatusIndicator` already uses this for unread. No new token needed.
- **Status orange (tool-input):** `oklch(0.75 0.18 55)` -- Custom amber/orange for tool confirmation waiting state. Define as `--status-tool-input` CSS variable in `:root` and `.dark`. Dark mode value: `oklch(0.78 0.16 60)`.
- **Status gray (streaming):** `var(--sidebar-primary)` with `animate-pulse` -- Existing pattern, pulsing distinguishes from solid unread dot.
- **Status red (error):** `var(--destructive)` -- Existing pattern.

### Concurrency Warning Banner Colors

- **Background:** `oklch(0.95 0.04 85)` (light) / `oklch(0.30 0.04 85)` (dark) -- Warm amber tint. Define as `--warning-bg` CSS variable.
- **Foreground:** `oklch(0.45 0.12 55)` (light) / `oklch(0.85 0.08 70)` (dark) -- Define as `--warning-fg` CSS variable.
- **Border:** `oklch(0.85 0.08 75)` (light) / `oklch(0.40 0.06 75)` (dark) -- Define as `--warning-border` CSS variable.

## Component Inventory

### Modified Components

**SessionStatusIndicator** (`src/components/session-status-indicator.tsx`)
- Extend `SessionStatus` type from `"streaming" | "unread" | "error" | "idle"` to include `"tool-input"`
- Add orange dot class: `status === "tool-input" && "bg-[var(--status-tool-input)]"`
- Priority order for display (only one dot shown): error > tool-input > unread > streaming > idle (no dot)
- Dot size remains `size-1.5` (6px diameter), `rounded-full`

**Chat area scroll container** (`src/routes/index.tsx`)
- Replace current always-scroll-to-bottom behavior with smart scroll
- Track scroll position per tab via `scrollPositionAtom(tabId)` and `autoScrollAtom(tabId)` atoms
- Auto-scroll threshold: user is "at bottom" when `scrollHeight - scrollTop - clientHeight < 32px`
- On tab switch: restore saved scroll position from atom, do NOT scroll to bottom
- On new content while auto-scroll enabled: scroll to bottom instantly (`behavior: "instant"`)
- On new content while auto-scroll disabled: do not scroll
- Scrolling to bottom (within 32px threshold) re-enables auto-scroll AND clears unread for that tab

**Chat input** (`src/routes/index.tsx`)
- Replace TanStack Form local state with `draftInputAtom(tabId)` per-tab atom
- Input value reads from and writes to `draftInputAtom(tabKey)`
- On submit: clear the draft atom, dispatch `sendMessageAtom`
- Preserves typed text when switching tabs and returning

### New Components

**ConcurrencyWarningBanner** (new: `src/components/concurrency-warning-banner.tsx`)
- Inline banner at the top of `SidebarContent` (before project groups)
- Renders when `concurrentStreamCountAtom` value >= 5
- Auto-hides when count drops below 5 (reactive)
- Layout: single-line banner, 8px vertical padding, 16px horizontal padding
- Typography: 12px (`text-xs`), weight 400
- Icon: warning/alert icon from hugeicons on the left, 16px size
- No dismiss button -- auto-dismisses based on stream count
- Copy: "Performance may degrade with {N} active streams."
- Uses `--warning-bg`, `--warning-fg`, `--warning-border` CSS variables
- Border: 1px solid, `rounded-md`

### New Atoms

- `unreadAtom`: `Atom.family((tabId: string) => Atom.make(false))` with `Atom.keepAlive`
- `toolInputAtom`: `Atom.family((tabId: string) => Atom.make(false))` with `Atom.keepAlive`
- `scrollPositionAtom`: `Atom.family((tabId: string) => Atom.make(0))` with `Atom.keepAlive`
- `autoScrollAtom`: `Atom.family((tabId: string) => Atom.make(true))` with `Atom.keepAlive`
- `draftInputAtom`: `Atom.family((tabId: string) => Atom.make(""))` with `Atom.keepAlive`
- `concurrentStreamCountAtom`: Derived atom that counts tabs where `isStreamingAtom` is true
- Extended `tabStatusAtom`: Derives `"error" | "tool-input" | "unread" | "streaming" | "idle"` from `isStreamingAtom`, `errorAtom`, `toolInputAtom`, `unreadAtom`

## Interaction Contracts

### Background Streaming (D-01, D-02)

- Streams continue running when user switches tabs
- Atom state updates regardless of which tab component is mounted (effect-atom handles this)
- No pause/resume logic -- fibers run independently via `sendMessageAtom`
- When user switches back to a tab, they see all accumulated content immediately

### Status Indicator Priority (D-03 through D-06)

Single dot displayed per session entry. Priority (highest wins):
1. **Error** (solid red `var(--destructive)`) -- session hit an error user hasn't retried
2. **Tool input** (solid orange `var(--status-tool-input)`) -- Claude waiting for tool confirmation
3. **Unread** (solid `var(--sidebar-primary)`) -- completed response user hasn't scrolled to
4. **Streaming** (pulsing `var(--sidebar-primary)` with `animate-pulse`) -- actively responding
5. **Idle** -- no dot rendered

### Unread Tracking (D-07 through D-09)

- Mark unread: Set `unreadAtom(tabId)` to `true` when `AssistantMessageEvent` arrives AND `activeTabIdAtom !== tabId`
- Clear unread: Set `unreadAtom(tabId)` to `false` when user scrolls to bottom of that tab's chat (within 32px threshold)
- Partial streaming content does NOT trigger unread
- Tab click alone does NOT clear unread

### Error Persistence (D-09)

- Error indicator persists until user sends a new message (which resets `errorAtom` to null)
- Error does NOT clear on tab view or tab click

### Concurrency Warning (D-10 through D-12)

- Warning threshold: 5 concurrent streams
- Display: Inline banner at top of sidebar content
- Behavior: Reactive -- appears when count >= 5, disappears when count < 5
- Non-blocking -- user can continue working

### Tab Switch (D-13 through D-16)

- **Scroll preservation (D-13):** Save `scrollTop` to `scrollPositionAtom(tabId)` on scroll events (debounced). Restore on tab switch.
- **Smart auto-scroll (D-14):** Auto-scroll ON when at bottom. Disable on scroll up. Re-enable when user scrolls back to bottom (within 32px).
- **Instant swap (D-15):** No animation or transition. Content swap is immediate on tab click.
- **Draft preservation (D-16):** Per-tab `draftInputAtom` preserves typed input text across tab switches.

### Tool Input Detection

- Detect tool confirmation events from the Claude CLI event stream
- Set `toolInputAtom(tabId)` to `true` when a tool_use content block is received without a subsequent tool_result
- Clear `toolInputAtom(tabId)` when streaming resumes or user sends a message
