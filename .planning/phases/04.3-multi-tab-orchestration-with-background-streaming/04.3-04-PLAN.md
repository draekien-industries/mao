  <action>
Extract the chat body into a separate `ChatPanel` component so that atom family hooks are never called with a "none" key.

**Why:** When activeTabId is null (before DB load completes), `String(activeTabId ?? "none")` produces "none" as tabKey. All atom subscriptions (messagesAtom, unreadAtom, etc.) key to "none", which never matches `tabStatusAtom(String(session.id))` in the sidebar. The unread flag set by sendMessageAtom lands on `unreadAtom("none")` instead of `unreadAtom("42")`.

**Implementation:**

1. Split `IndexComponent` into two parts:
   - `IndexComponent` (outer): reads `activeTabIdAtom`. If null, renders placeholder. If non-null, renders `<ChatPanel tabKey={String(activeTabId)} />`.
   - `ChatPanel` (inner): accepts `{ tabKey: string }` prop. Contains all the existing hooks and JSX that currently live in IndexComponent (everything after the `tabKey` assignment).

2. The outer component:
   ```tsx
   function IndexComponent() {
     const activeTabId = useAtomValue(activeTabIdAtom);
     if (activeTabId === null) {
       return (
         <div className="flex h-full items-center justify-center">
           <p className="text-sm text-muted-foreground">
             Select a session to start chatting.
           </p>
         </div>
       );
     }
     return <ChatPanel tabKey={String(activeTabId)} />;
   }
   ```

3. The `ChatPanel` component receives `tabKey` as a prop and uses it for all atom family subscriptions. This is safe because it only mounts when activeTabId is a real number, so tabKey is always `String(numericId)`.

4. All existing hooks (useAtomValue for messages, streaming, scroll, draft, etc.) move into ChatPanel unchanged. The early return in IndexComponent is before any hooks except `useAtomValue(activeTabIdAtom)`, so there are no hooks-order violations.

**IMPORTANT per D-08:** Do NOT clear unreadAtom on tab switch. The existing onScroll handler (lines 55-77) already clears unread via `setUnread(false)` when the user scrolls to the bottom. This is the correct behavior -- unread clears on scroll-to-bottom, not on tab click.
