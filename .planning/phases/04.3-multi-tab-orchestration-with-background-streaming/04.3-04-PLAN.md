Two changes are required:

1. **src/routes/index.tsx** — Guard against null activeTabId when computing tabKey.

   Currently line 32 reads:
   ```ts
   const tabKey = String(activeTabId ?? "none");
   ```

   The problem: when activeTabId is null (before DB load completes), all atom subscriptions and sendMessage calls key to "none", which never matches `tabStatusAtom(String(session.id))` in the sidebar. The unread flag set by sendMessageAtom lands on `unreadAtom("none")` instead of `unreadAtom("42")`.

   Fix: Early-return a placeholder/empty UI when activeTabId is null. The component should render a centered "Send a message to start chatting" placeholder (or nothing) when there is no active tab, and only proceed with atom subscriptions when activeTabId is a real number. This ensures tabKey is always `String(numericId)` and never `"none"`.

   Implementation approach:
   ```ts
   const activeTabId = useAtomValue(activeTabIdAtom);
   if (activeTabId === null) {
     return (
       <div className="flex h-full items-center justify-center">
         <p className="text-sm text-muted-foreground">Select a session to start chatting.</p>
       </div>
     );
   }
   const tabKey = String(activeTabId);
   ```

   Move the null check BEFORE any atom family subscriptions so React hooks are not called conditionally — the early return must come after `useAtomValue(activeTabIdAtom)` but before all other hooks. Since hooks cannot be called conditionally, extract the body that uses tabKey into a separate component (e.g., `ChatPanel`) that only mounts when activeTabId is non-null. The outer `IndexComponent` does the null check and either renders the placeholder or `<ChatPanel tabKey={String(activeTabId)} />`.

2. **src/atoms/sidebar.ts** — Clear unreadAtom when switching to a tab.

   Currently `setActiveTabAtom` (line 70-78) sets activeTabIdAtom but never clears the unread flag on the newly activated tab.

   Fix: Import `unreadAtom` from `@/atoms/chat` and clear it in `setActiveTabAtom`:
   ```ts
   import { unreadAtom } from "@/atoms/chat";

   export const setActiveTabAtom = appRuntime.fn(
     (tabId: number, ctx: Atom.FnContext) =>
       Effect.sync(() => {
         ctx.set(sessionLoadingAtom, true);
         ctx.set(activeTabIdAtom, tabId);
         ctx.set(unreadAtom(String(tabId)), false);
         ctx.set(sessionLoadingAtom, false);
       }),
   );
   ```

   This ensures the green dot disappears immediately on tab switch, even before the user scrolls.
