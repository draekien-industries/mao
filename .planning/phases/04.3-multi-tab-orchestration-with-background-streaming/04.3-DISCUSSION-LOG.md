## Unread Tracking

### When should a tab become "unread"?

| Option | Description | Selected |
|--------|-------------|----------|
| Stream completes | Tab marked unread only when a full assistant response finishes while the tab is not visible | ✓ |
| Any new content | Tab marked unread as soon as any streaming content arrives while the tab is not visible | |

**User's choice:** Stream completes
**Notes:** Partial streaming doesn't trigger unread — only complete AssistantMessageEvent.

### How should the unread state clear?

| Option | Description | Selected |
|--------|-------------|----------|
| Tab click | Unread clears the moment the user clicks the tab in the sidebar | |
| Tab focus + delay | Unread clears after the tab has been visible for ~1 second | |
| You decide | Let Claude choose the clearing mechanism | |

**User's choice:** Initially selected "Tab click", but later refined during Tab switch behavior discussion to "scroll to bottom" — unread clears when user scrolls to the bottom of the chat, not on tab click.
**Notes:** This decision was refined when discussing scroll preservation behavior. The final decision ties unread clearing to actually seeing the content.

### Error and unread overlap?

| Option | Description | Selected |
|--------|-------------|----------|
| Error supersedes | Error indicator replaces unread badge. User sees red error dot only. | ✓ |
| Both shown | Show both indicators simultaneously | |
| You decide | Let Claude choose priority behavior | |

**User's choice:** Error supersedes
**Notes:** Error is more actionable, so it takes priority.

### Error indicator clearing behavior?

| Option | Description | Selected |

## Tab Switch Behavior

### Auto-scroll on tab switch?

| Option | Description | Selected |
|--------|-------------|----------|
| Always auto-scroll | Always scrolls to bottom on switch | |
| Preserve scroll position | Remembers position per tab | ✓ |
| Smart scroll | Auto-scroll if was at bottom, preserve if scrolled up | |

**User's choice:** Preserve scroll position, with smart scroll refinement
**Notes:** User provided detailed specification: "Preserve scroll position. Switching back puts the user where they left off. Once the user scrolls down to the bottom it clears the unread status and we switch back to auto-scroll. If the user at any time scrolls up from the bottom we disable auto-scroll." This also refined the unread clearing mechanism from "tab click" to "scroll to bottom".
