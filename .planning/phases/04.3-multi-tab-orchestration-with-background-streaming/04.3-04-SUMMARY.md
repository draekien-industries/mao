## What Was Done

### Task 1: Extract ChatPanel to guard null activeTabId

Split `IndexComponent` into two components:

1. **IndexComponent (outer):** Reads `activeTabIdAtom`. When null, renders a centered placeholder message. When non-null, renders `<ChatPanel tabKey={String(activeTabId)} />`.

2. **ChatPanel (inner):** Accepts `{ tabKey: string }` prop with a guaranteed-valid numeric ID string. Contains all existing hooks (messages, streaming, scroll, draft, unread) and JSX. Only mounts when a real tab is active.

**Root cause fixed:** Previously `String(activeTabId ?? "none")` produced a "none" tabKey when activeTabId was null (before DB load). All atom family subscriptions keyed to "none" which never matched `tabStatusAtom(String(session.id))` in the sidebar. The unread flag set by `sendMessageAtom` landed on `unreadAtom("none")` instead of the real tab's atom instance.

**D-08 preserved:** The existing `onScroll` handler clears unread via `setUnread(false)` when the user scrolls to bottom. No changes to unread-clearing behavior were made.
