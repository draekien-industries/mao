# Phase 5 — UI Design Contract

> Visual and interaction contract for renderer integration. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

- **Tool:** shadcn v4.1.0
- **Preset:** base-maia
- **Component library:** base-ui/react (via shadcn)
- **Icon library:** hugeicons (core-free-icons)
- **Font:** System default (inherited from shadcn neutral theme)

---

## Spacing Scale

Declared values (multiples of 4, consistent with existing codebase):

- **xs (4px):** Icon gaps, inline padding
- **sm (8px):** Compact element spacing, skeleton block gaps
- **md (16px):** Default element spacing, message padding (existing `p-4`)
- **lg (24px):** Section padding
- **xl (32px):** Layout gaps
- **2xl (48px):** Major section breaks
- **3xl (64px):** Page-level spacing

Exceptions: none

---

## Typography

- **Body:** 14px, weight 400, line-height 1.5 (existing `text-sm` throughout chat panel)
- **Label:** 14px, weight 600, line-height 1.5 (sidebar labels, status text)
- **Heading:** 20px, weight 600, line-height 1.2 (not new this phase -- no new headings introduced)
- **Caption:** 12px, weight 400, line-height 1.5 (tool result metadata, timestamps)

Source: Pre-populated from existing chat panel (`text-sm` = 14px body, `text-xs` = 12px for error/status).

---

## Color

- **Dominant (60%):** `var(--background)` -- Main chat area background, page surfaces
- **Secondary (30%):** `var(--muted)` -- Assistant message bubbles, skeleton placeholder blocks, tool result backgrounds
- **Accent (10%):** `var(--primary)` -- User message bubbles, send button, active tab indicator
- **Destructive:** `var(--destructive)` -- Error state banners only

Accent reserved for: user message bubble background, send button, primary CTA interactions. Never for tool result blocks or skeleton elements.

Source: Pre-populated from existing `index.tsx` chat panel color usage (bg-primary for user, bg-muted for assistant, bg-destructive/10 for errors).

## Interaction States

### Session Hydration Loading (D-03)

- **Trigger:** App start (active tab) or tab click (lazy load)
- **Visual:** 3 alternating MessageSkeleton blocks (user-right, assistant-left, user-right) with staggered widths
- **Duration:** Shown while `sessionLoadingAtom` is true; resolves into real messages
- **Transition:** Skeleton blocks fade out, real messages appear (no layout shift -- skeletons match message dimensions)

### MessageSkeleton Specification

- User skeleton: right-aligned, `bg-primary/20` rounded-2xl block, width 40-60%, height 36px
- Assistant skeleton: left-aligned, `bg-muted` rounded-2xl block, width 60-80%, height 52px
- Spacing: `space-y-3` (matches existing message gap)
- Animation: Use shadcn `Skeleton` pulse animation (already in codebase)

### Tool Result Block Specification (D-10)

- Position: Between assistant message (with tool_use) and next assistant message
- Background: `bg-muted/50` (lighter than assistant messages to visually distinguish)
- Border: `border border-border` with `rounded-lg`
- Padding: `px-3 py-2` (matches message padding)
- Layout: Left-aligned, max-width 80% (matches assistant messages)
- Label: "Tool Result" in `text-xs font-semibold text-muted-foreground` above content
- Content: Preformatted text in `text-sm whitespace-pre-wrap`
- Error indicator: If `is_error` is true, label changes to "Tool Error" with `text-destructive` color

### Restored Conversation (D-11)

- Restored messages appear as static history (identical to live messages visually)
- No streaming cursor on restored messages
- No auto-scroll on load -- scroll starts at top of conversation
- User sends a new message to resume (uses `--resume` flag under the hood)

### Empty State (no changes)

- When `activeTabId` is null: "Select a session to start chatting." (existing)
- When tab has no messages and no hydration in progress: "Send a message to start chatting." (existing)

### Error State (no changes)

- Existing: `bg-destructive/10 text-destructive` banner below messages, above input
- Copy format: "[Error description from formatClaudeCliError]"
- No new error states introduced this phase
