    Create `src/services/claude-cli/persistent/service.ts` implementing the decorator.

    Structure:

    ```
    import { Effect, Layer, Stream, Ref } from "effect";
    import crypto from "node:crypto";
    import { isAssistantMessage, isResult, isSystemInit } from "../events";
    import type { ClaudeEvent } from "../events";
    import type { ClaudeCliError } from "../errors";
    import { QueryParams } from "../params";
    import { ClaudeCli } from "../service-definition";
    import { EventStore } from "../../database/event-store/service-definition";
    import type { DatabaseQueryError } from "../../database/errors";
    import { annotations } from "../../diagnostics";
    ```

    1. `persistEvent` helper function:
       - Takes `store` (EventStore service value, typed as `Context.Tag.Service<typeof EventStore>`), `sessionId: string`, `eventType: string`, `event: ClaudeEvent`
       - Calls `store.append(sessionId, eventType, JSON.stringify(event))`
       - Pipes through `Effect.catchAll((err: DatabaseQueryError) => Effect.logWarning("Persistence write failed").pipe(Effect.annotateLogs("error", err.message), Effect.annotateLogs(annotations.sessionId, sessionId), Effect.annotateLogs("eventType", eventType)))` (per D-18)

    2. `persistUserMessage` helper function:
       - Takes `store`, `sessionId: string`, `prompt: string`
       - Calls `store.append(sessionId, "user_message", JSON.stringify({ type: "user_message", prompt }))`
       - Same `Effect.catchAll` pattern for write failure swallowing

    3. `wrapStream` helper function:
       - Takes `store`, `sessionId: string`, `stream: Stream.Stream<ClaudeEvent, ClaudeCliError, never>`
       - Returns `stream.pipe(Stream.tap((event) => { ... }))` with conditional persistence:
         - `if (isSystemInit(event))` -> `persistEvent(store, sessionId, "system", event)`
         - `if (isAssistantMessage(event))` -> `persistEvent(store, sessionId, "assistant", event)`
         - `if (isResult(event))` -> `persistEvent(store, sessionId, "result", event)`
         - else -> `Effect.void` (D-04, D-05, D-06: discard StreamEventMessage, SystemRetryEvent, UnknownEvent)

    4. `makePersistentClaudeCliLive` exported function:
       - Returns `Layer.effect(ClaudeCli, Effect.gen(function* () { ... }).pipe(Effect.annotateLogs(annotations.service, "persistent-cli")))`
       - Inside the generator:
         - `const inner = yield* ClaudeCli` (gets the underlying ClaudeCli implementation)
         - `const store = yield* EventStore` (gets the EventStore)
         - `yield* Effect.logInfo("PersistentClaudeCliLive layer constructed")`
         - Returns object with `query`, `resume`, `cont` methods:

       **query(params):**
       - `const sessionId = crypto.randomUUID()` (D-08)
       - `const enrichedParams = new QueryParams({ ...params, session_id: sessionId })` — injects pre-generated ID
       - Pre-stream: persist user message via `Stream.fromEffect(persistUserMessage(store, sessionId, params.prompt)).pipe(Stream.drain)`
       - `Stream.concat(wrapStream(store, sessionId, inner.query(enrichedParams)))` — wraps inner stream with persistence

       **resume(params):**
       - `const sessionId = params.session_id` (already known from ResumeParams)
       - Pre-stream: persist user message same pattern
       - `Stream.concat(wrapStream(store, sessionId, inner.resume(params)))` — wraps inner stream

       **cont(params):**
       - Session ID is unknown upfront (ContinueParams has no session_id field)
       - Use `Ref.make("")` to create a mutable ref for session_id
       - Wrap the inner stream with a single `Stream.tap` that:
         - On SystemInitEvent: sets the ref to event.session_id, persists the SystemInitEvent, then persists the user message (D-09 adapted for cont: user message comes after SystemInitEvent)
         - On AssistantMessageEvent: reads ref, persists if ref is non-empty
         - On ResultEvent: reads ref, persists if ref is non-empty
         - All others: Effect.void
       - Return `inner.cont(params).pipe(Stream.tapEffect(...))` — note: for cont, we do NOT use the wrapStream helper since the session_id flow is different
