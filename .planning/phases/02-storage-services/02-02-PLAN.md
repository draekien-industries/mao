Follow the exact pattern from `src/services/database/service.ts` (Layer.effect with Database dependency):

```typescript
import { SqlClient } from "@effect/sql";
import { Effect, Layer, Schema } from "effect";
import { annotations } from "../../diagnostics";
import { DatabaseQueryError } from "../errors";
import { Database } from "../service-definition";
import { StoredEvent } from "./schemas";
import { EventStore } from "./service-definition";

interface EventRow {
  readonly id: number;
  readonly session_id: string;
  readonly sequence_number: number;
  readonly event_type: string;
  readonly event_data: string;
  readonly created_at: string;
}

export const makeEventStoreLive = () =>
  Layer.effect(
    EventStore,
    Effect.gen(function* () {
      const { sql } = yield* Database;

      const append = (
        sessionId: string,
        eventType: string,
        eventData: string,
      ) =>
        sql`
          INSERT INTO events (session_id, sequence_number, event_type, event_data)
          VALUES (
            ${sessionId},
            (SELECT COALESCE(MAX(sequence_number), 0) + 1
             FROM events WHERE session_id = ${sessionId}),
            ${eventType},
            ${eventData}
          )
        `.pipe(
          Effect.asVoid,
          Effect.mapError(
            (cause) =>
              new DatabaseQueryError({
                cause: String(cause),
                message: `Failed to append event for session ${sessionId}`,
              }),
          ),
          Effect.annotateLogs(annotations.operation, "append"),
        );

      const getBySession = (sessionId: string) =>
        Effect.gen(function* () {
          const rows = yield* sql<EventRow>`
            SELECT id, session_id, sequence_number, event_type, event_data, created_at
            FROM events
            WHERE session_id = ${sessionId}
            ORDER BY sequence_number ASC
          `;

          const decoded: StoredEvent[] = [];
          for (const row of rows) {
            const event = yield* Schema.decodeUnknown(StoredEvent)(
              JSON.parse(row.event_data),
            ).pipe(
              Effect.mapError(
                (cause) =>
                  new DatabaseQueryError({
                    cause: String(cause),
                    message: `Failed to decode event ${row.id} for session ${sessionId}`,
                  }),
              ),
            );
            decoded.push(event);
          }
          return decoded as ReadonlyArray<StoredEvent>;
        }).pipe(
          Effect.mapError(
            (cause) =>
              cause instanceof DatabaseQueryError
                ? cause
                : new DatabaseQueryError({
                    cause: String(cause),
                    message: `Failed to query events for session ${sessionId}`,
                  }),
          ),
          Effect.annotateLogs(annotations.operation, "getBySession"),
        );

      const purgeSession = (sessionId: string) =>
        sql`DELETE FROM events WHERE session_id = ${sessionId}`.pipe(
          Effect.asVoid,
          Effect.mapError(
            (cause) =>
              new DatabaseQueryError({
                cause: String(cause),
                message: `Failed to purge events for session ${sessionId}`,
              }),
          ),
          Effect.annotateLogs(annotations.operation, "purgeSession"),
        );

      return { append, getBySession, purgeSession };
    }).pipe(Effect.annotateLogs(annotations.service, "event-store")),
  );
```

**Key implementation details:**
- `append`: Uses subselect `COALESCE(MAX(sequence_number), 0) + 1` for atomic sequence number assignment per session (EVNT-04). No caller-provided sequence number. Per D-03, stores raw eventData string without Schema validation.
- `getBySession`: Queries by session_id with ORDER BY sequence_number ASC (EVNT-03, EVNT-04). Decodes each row's event_data JSON via `Schema.decodeUnknown(StoredEvent)` per D-03. Maps decode errors to DatabaseQueryError.
- `purgeSession`: Simple DELETE WHERE session_id = X per D-08.
- All methods annotate logs with operation name. The outer generator annotates with service name.
- Use `Effect.asVoid` after INSERT/DELETE to discard the result rows.

**Important:** The `getBySession` error handling needs care. The outer `Effect.mapError` should only wrap non-DatabaseQueryError causes (the SQL query failure). The inner decode errors are already wrapped. Use a conditional check: if the error is already a DatabaseQueryError, pass it through; otherwise wrap it.
