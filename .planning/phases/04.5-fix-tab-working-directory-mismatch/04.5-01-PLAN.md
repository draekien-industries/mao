<objective>
Thread each tab's working directory through to the Claude CLI process spawn, fixing the bug where all CLI processes start in the app's default directory instead of the tab's project directory.

Purpose: Without this fix, Claude CLI operations (file reads, tool use, context) target the wrong directory for every tab, making multi-project support broken in practice.
Output: `cwdAtom` family in chat.ts, population in all sidebar tab activation paths, cwd passed to QueryParams/ResumeParams in sendMessageAtom, tests.
</objective>
