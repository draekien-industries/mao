# Phase 04.2 -- UI Design Contract

> Visual and interaction contract for project registration, session creation, and git integration flows. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

- **Tool:** shadcn v4.1.0
- **Preset:** base-maia (style: base-maia, base color: neutral)
- **Component library:** base-ui (via @base-ui/react)
- **Icon library:** hugeicons (@hugeicons/react + @hugeicons/core-free-icons)
- **Font:** System font stack (Tailwind default -- no custom font loaded)

Source: Inherited from Phase 04.1 UI-SPEC. No changes.

---

## Spacing Scale

Inherited from Phase 04.1. Same 8-point scale.

- **xs (4px):** Icon gaps, inline padding, checkbox-to-label gap
- **sm (8px):** Dialog field vertical gap between label and input, compact element spacing
- **md (16px):** Dialog body padding (`p-4`), form field vertical spacing between groups, context menu item padding
- **lg (24px):** Dialog header/footer padding, section breaks within dialogs
- **xl (32px):** Not actively used in this phase
- **2xl (48px):** Not actively used in this phase
- **3xl (64px):** Empty state vertical padding

Exceptions: Dialog max-width is `28rem` (448px) for session creation dialog and confirmation dialog. Native directory picker is OS-controlled (no sizing contract).

---

## Typography

Inherited from Phase 04.1. Two weights: 400 (regular) and 600 (semibold).

- **Body (14px / text-sm):** Weight 400, line-height 1.43 (20px). Dialog body text, input values, branch autocomplete items, context menu items.
- **Label (12px / text-xs):** Weight 400, line-height 1.33 (16px). Form field labels, checkbox labels, helper text below inputs, worktree path display.
- **Heading (16px / text-base):** Weight 600, line-height 1.5 (24px). Dialog title text ("New Session", "Remove Project", "Register Project").
- **Display:** Not used in this phase.

---

## Color

Inherited from Phase 04.1. All colors use shadcn CSS custom properties in OKLCH.

- **Dominant (60%) -- `var(--background)`:** Dialog backdrop overlay uses `var(--background)` at 80% opacity (the dialog component default).
- **Secondary (30%) -- `var(--card)` / `var(--muted)`:** Dialog surface (`var(--card)`), input backgrounds, autocomplete dropdown surface.
- **Accent (10%) -- `var(--primary)` / `var(--sidebar-accent)`:** Reserved for: "Create Session" primary CTA button (`bg-primary`), "Register Project" primary CTA button, active autocomplete item highlight, "+" button on project group header.
- **Destructive -- `var(--destructive)`:** "Remove Project" confirmation button (`bg-destructive`), worktree removal confirmation button, error messages in dialog forms (validation failures).

Accent reserved for: primary CTA buttons in dialogs, "+" add-session button in sidebar, active autocomplete selection highlight.

## Component Inventory

### New shadcn components to install

- **dialog:** Required for session creation modal (D-01, D-05), project removal confirmation (D-03), and worktree reuse prompt (D-12). Provides `Dialog`, `DialogTrigger`, `DialogContent`, `DialogHeader`, `DialogTitle`, `DialogDescription`, `DialogFooter`, `DialogClose`.
- **checkbox:** Required for "Use worktree" toggle in session creation dialog (D-10, D-11). Provides `Checkbox`.
- **label:** Required for form field labels in dialogs. Provides `Label`.
- **context-menu:** Required for right-click project removal (D-02). Provides `ContextMenu`, `ContextMenuTrigger`, `ContextMenuContent`, `ContextMenuItem`.
- **command:** Required for branch name autocomplete in session creation dialog (D-10). Provides `Command`, `CommandInput`, `CommandList`, `CommandEmpty`, `CommandGroup`, `CommandItem`.

### Existing shadcn components reused

- **sidebar:** All sidebar primitives from Phase 04.1 (SidebarGroup, SidebarGroupLabel, SidebarGroupAction, SidebarMenu, SidebarMenuItem, SidebarMenuButton)
- **button:** Dialog CTA buttons, "+" add session button in sidebar group header
- **input:** Session name/branch input field in creation dialog, worktree base path input
- **skeleton:** Loading transition when switching sessions (D-04)
- **collapsible:** Project group expand/collapse (inherited)
- **dropdown-menu:** Already installed, available as fallback if context-menu presents issues

### New app components to create

- **CreateSessionDialog:** Modal for creating a new session within a project. Contains: branch name input with autocomplete, "Use worktree" checkbox, worktree base path display, "Create Session" CTA. Props: `projectId: string`, `isGitRepo: boolean`, `open: boolean`, `onOpenChange: (open: boolean) => void`. Location: `src/components/create-session-dialog.tsx`
- **RemoveProjectDialog:** Confirmation modal for project removal. Shows project name and session count. Contains: "Remove" destructive CTA and "Cancel" secondary CTA. Props: `projectName: string`, `sessionCount: number`, `open: boolean`, `onOpenChange: (open: boolean) => void`, `onConfirm: () => void`. Location: `src/components/remove-project-dialog.tsx`
- **RemoveWorktreeDialog:** Confirmation modal when removing a session with an associated worktree (D-14). Props: `worktreePath: string`, `open: boolean`, `onOpenChange: (open: boolean) => void`, `onConfirm: (removeWorktree: boolean) => void`. Location: `src/components/remove-worktree-dialog.tsx`
- **ProjectContextMenu:** Right-click context menu wrapping project group headers. Contains: "Remove project" item. Props: `children: React.ReactNode`, `onRemove: () => void`. Location: `src/components/project-context-menu.tsx`
- **BranchAutocomplete:** Input with command-based autocomplete dropdown for git branch names. Filters as user types. Shows "No branches found" when empty. Props: `branches: string[]`, `value: string`, `onChange: (value: string) => void`, `disabled: boolean`. Location: `src/components/branch-autocomplete.tsx`

## Layout Contract

### Focal point

- **Primary visual anchor:** The "+" button on each project group header in the sidebar. This is the entry point for the main workflow of this phase (session creation). It uses `SidebarGroupAction` positioning (right-aligned in the group label row).
- **Secondary visual anchor:** The newly created session entry appearing in the sidebar after creation, with active state highlighting.

### Sidebar modifications

```
SidebarContent (scrollable)
  For each project:
    ContextMenu
      ContextMenuTrigger
        Collapsible (defaultOpen)
          SidebarGroup
            SidebarGroupLabel (project name, CollapsibleTrigger)
              Chevron icon (rotate on collapse)
              Project name (truncated)
              SidebarGroupAction: "+" button (opens CreateSessionDialog)
            CollapsibleContent
              SidebarGroupContent
                SidebarMenu
                  For each session:
                    SidebarMenuItem
                      SidebarMenuButton (isActive, onClick -> switch tab)
                        Branch label (truncated)
                        SessionStatusIndicator
      ContextMenuContent
        ContextMenuItem: "Remove project"
```

### Session creation dialog layout

```
Dialog (max-width: 28rem)
  DialogHeader
    DialogTitle: "New Session"
    DialogDescription: "Create a session in {projectName}"
  DialogContent (body)
    Field group: Branch name
      Label: "Branch"
      BranchAutocomplete (with command dropdown)
    Field group: Worktree option (visible only if isGitRepo)
      Checkbox + Label: "Create worktree"
      Helper text (12px muted): "Isolates work in a separate directory"
      Worktree path preview (12px muted): "{basePath}/{branchName}"
  DialogFooter
    Button (variant="outline"): "Cancel"
    Button (variant="default"): "Create Session"
```

### Remove project confirmation dialog layout

```
Dialog (max-width: 28rem)
  DialogHeader
    DialogTitle: "Remove Project"
    DialogDescription: "This will close {N} session(s) in {projectName}."
  DialogFooter
    Button (variant="outline"): "Cancel"
    Button (variant="destructive"): "Remove"
```

### Remove worktree confirmation dialog layout

```
Dialog (max-width: 28rem)
  DialogHeader
    DialogTitle: "Remove Session"
    DialogDescription: "This session has a worktree at:"
  DialogContent (body)
    Code block (12px monospace, muted bg): "{worktreePath}"
  DialogFooter
    Button (variant="outline"): "Keep worktree"
    Button (variant="destructive"): "Remove worktree too"
```

### Session loading transition (D-04)

When a session is clicked in the sidebar and its atom state is not yet resolved:
- Chat panel shows 3 skeleton lines (varying widths: 75%, 60%, 85%) stacked vertically with `sm` (8px) gaps
- Skeleton uses `bg-muted animate-pulse` (shadcn skeleton default)
- Duration: display until atom state resolves, typically under 200ms

## Copywriting Contract

- **Primary CTA (session creation):** "Create Session"
- **Primary CTA (project registration):** "Register Project" (used if a registration dialog is shown; otherwise the native directory picker is the entry point)
- **Sidebar "+" button:** aria-label: "New session" (icon-only button, no visible text)
- **Empty state heading (no projects):** "No projects yet"
- **Empty state body (no projects):** "Register a project folder to get started." with a "Register Project" button below.
- **Empty state (no sessions in a project):** Not reachable -- D-17 auto-creates a first session on registration.
- **Error state (git operation failed):** "Git operation failed. Check that git is installed and the directory is a valid repository."
- **Error state (worktree creation failed):** "Could not create worktree. The branch may already have a worktree, or the path is not writable."
- **Error state (branch listing failed):** Autocomplete shows "Could not load branches" in the empty state position.
- **Destructive: Remove project:** Dialog title: "Remove Project". Description: "This will close {N} session(s) in {projectName}." CTA: "Remove" (destructive variant).
- **Destructive: Remove worktree:** Dialog title: "Remove Session". Description: "This session has a worktree at:". Options: "Keep worktree" (outline) / "Remove worktree too" (destructive).
- **Worktree reuse prompt (D-12):** Dialog title: "Worktree Exists". Description: "A worktree for '{branchName}' already exists at {path}." CTA: "Use Existing" (default variant) / "Cancel" (outline).
- **Checkbox label:** "Create worktree"
- **Checkbox helper text:** "Isolates work in a separate directory"
- **Worktree path preview:** "{basePath}/{branchName}" (shown below checkbox when checked)
- **Branch autocomplete empty:** "No branches found"
- **Branch autocomplete placeholder:** "Search or enter branch name..."
- **Session name input placeholder:** Derived from branch name automatically, no separate input needed.
- **Non-git project notice:** Helper text on session creation dialog when `isGitRepo` is false: "Git features unavailable for this project."

## Interaction States

### "+" (add session) button in sidebar group header

- **Default:** `text-sidebar-foreground/70`, icon only (use Add01Icon or equivalent from hugeicons)
- **Hover:** `text-sidebar-foreground`
- **Click:** Opens CreateSessionDialog with the parent project context

### Project group header (right-click)

- **Default:** Same as Phase 04.1 -- text-xs, font-normal, collapsible trigger
- **Right-click:** Opens ContextMenu with "Remove project" item
- **Context menu item hover:** `bg-accent text-accent-foreground` (shadcn default)
- **Context menu destructive item:** "Remove project" uses `text-destructive` color

### CreateSessionDialog form

- **Branch input default:** Empty, placeholder "Search or enter branch name..."
- **Branch input typing:** Autocomplete dropdown appears below input, filtered by typed text
- **Autocomplete item hover:** `bg-accent` highlight
- **Autocomplete item selected:** Value populates input, dropdown closes
- **Checkbox default (git repo):** Checked (D-11). Shows worktree path preview below.
- **Checkbox unchecked:** Worktree path preview hidden. Session will use branch checkout instead.
- **Checkbox (non-git project):** Hidden entirely (D-15).
- **Create Session button:** Disabled when branch name input is empty. Enabled when branch name has value.
- **Submitting:** Button shows loading state (disabled + "Creating..." text)

### RemoveProjectDialog

- **Open trigger:** Right-click context menu "Remove project" item
- **Cancel:** Closes dialog, no action
- **Remove:** Fires onConfirm, closes dialog, project removed from sidebar

### Session click (switching)

- **Click session entry:** Immediately highlight as active (`data-active`). Chat panel shows skeleton loading state. Once atom state resolves, skeleton replaced with chat content.
- **Already active session:** No-op on click.
