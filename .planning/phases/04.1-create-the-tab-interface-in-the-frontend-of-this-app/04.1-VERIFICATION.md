### Human Verification Required

The following items require visual inspection of the running app (`npm start`) because they involve CSS animations, visual layout rendering, and interactive UI behavior that cannot be asserted programmatically.

#### 1. Sidebar Layout and Two-Tiered Structure

**Test:** Start the app. Observe the main window layout.
**Expected:** A sidebar panel appears on the left side. It contains a header with "Mao" text. Below it are two collapsible sections labeled "mao" and "example-project". Under "mao" is one session entry labeled "main". Under "example-project" are two entries: "feature/auth" and "develop".
**Why human:** Visual component rendering, layout composition (flex, h-screen), and shadcn SidebarProvider behavior require a real browser viewport.

#### 2. Collapsible Group Expand/Collapse with Chevron Rotation

**Test:** Click the "mao" or "example-project" project header row.
**Expected:** The session entries below it collapse (hide), and the ArrowDown01Icon rotates -90 degrees. Clicking again re-expands and the icon rotates back. The animation is smooth via CSS transition.
**Why human:** The base-ui Collapsible `data-closed`/`data-open` attribute toggle and the Tailwind `group-data-[closed]/collapsible:-rotate-90 transition-transform` CSS animation cannot be verified without a rendered DOM.

#### 3. Active Session Highlight

**Test:** Observe the "main" session entry under "mao" versus the entries under "example-project".
**Expected:** The "main" entry has distinct active styling (background highlight via `SidebarMenuButton isActive={true}`). The other entries ("feature/auth", "develop") appear in normal/inactive state.
**Why human:** The visual distinction of the `isActive` prop styling on SidebarMenuButton requires human inspection.

#### 4. Live Streaming Status Indicator

**Test:** Type a message in the chat input and submit it. Observe the "main" session entry in the sidebar during the response.
**Expected:** While Claude is streaming a response, a small pulsing dot (6px, sidebar-primary color) appears next to the "main" branch label in the sidebar. When the response completes, the dot disappears. If an error occurs, the dot turns red.
**Why human:** CSS `animate-pulse` animation and real-time atom state update from a live Claude CLI session require visual observation.
