## Decisions Made
- FnContext is callable (ctx(atom)) not ctx.get(atom) -- the FnContext interface has no named get method, only set
- RegistryProvider from @effect-atom/atom-react takes an options object with children prop
- Kept AppRuntime export in runtime.ts for backward compatibility since use-claude-chat.ts still needs a runtime
- Used --legacy-peer-deps for installation since effect-atom declares react >=18 <20 but project uses React 19

**1. [Rule 1 - Bug] Fixed FnContext API usage (ctx.get -> ctx callable)**
- **Found during:** Task 2 (Create per-tab chat atoms)
- **Issue:** Plan specified Atom.set/Atom.get as Effect-based APIs and ctx.get(atom). FnContext has no get method -- it is callable directly.
- **Fix:** Used ctx(atom) for reads and ctx.set(atom, value) for writes
- **Files modified:** src/atoms/chat.ts
- **Verification:** npm run typecheck passes
- **Committed in:** 6816d87 (Task 2 commit)

**2. [Rule 3 - Blocking] Updated use-claude-chat.ts to remove deleted useRuntime import**
- **Found during:** Task 2 (Swap app provider)
- **Issue:** Removing useRuntime export from runtime.ts broke use-claude-chat.ts which still imports it
- **Fix:** Changed import to use AppRuntime directly (still exported for backward compat)
- **Files modified:** src/hooks/use-claude-chat.ts
- **Verification:** npm run typecheck and npm test pass
- **Committed in:** 6816d87 (Task 2 commit)
