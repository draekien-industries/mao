| Option | Description | Selected |

## Multi-session state architecture

| Option | Description | Selected |
|--------|-------------|----------|
| One hook instance per tab | Mount/unmount useClaudeChat per active tab | |
| All tabs always mounted | Hidden DOM, all hooks stay alive | |
| Centralized tab state manager | Single store holds all tab states | ✓ |

**User's choice:** Centralized tab state manager

**Follow-up: Implementation approach**

| Option | Description | Selected |
|--------|-------------|----------|
| React Context + useReducer | Pure React, Map<tabId, TabState> in reducer | |
| Effect Ref/SubscriptionRef | State in Effect layer via SubscriptionRef | |
| effect-atom | tim-smart/effect-atom — reactive atoms with Effect + React hooks | ✓ |

**User's choice:** effect-atom (https://github.com/tim-smart/effect-atom)
**Notes:** User suggested this library. Key features: Atom.family for per-tab state, Atom.runtime for layer integration, Atom.pull for stream consumption, Result type for status states.

## Tab creation and defaults

**Project registration:**

| Option | Description | Selected |
|--------|-------------|----------|
| Auto-grouped from tabs | Sessions auto-group by project on creation | |
| Explicit project registration | User registers projects first, then creates sessions within | ✓ |

**User's choice:** Explicit project registration

**How to register a project:**

| Option | Description | Selected |
|--------|-------------|----------|
| Native directory picker | Electron dialog.showOpenDialog | ✓ |
| Text input for path | User types/pastes path | |

**User's choice:** Native directory picker

**After registration:**

| Option | Description | Selected |
|--------|-------------|----------|
| Immediate creation | Auto-create first session on current branch | ✓ |
| Confirm step | Show detected info, let user edit label | |
| Just register | No auto-session | |

**User's choice:** Immediate creation with first session on current branch

**Session creation within project:**

| Option | Description | Selected |
|--------|-------------|----------|
| "+" button, immediate | Create session instantly, default branch label | ✓ |
| "+" button, form | Popover to pick branch/worktree and label | |

**User's choice:** "+" button on project group header. Default label is current branch if no tabs exist. Otherwise ask user for branch name and whether to create a branch or worktree.

**Worktree creation:**

| Option | Description | Selected |
|--------|-------------|----------|

**User's choice:** App creates worktrees via `git worktree add`

**Project management:**

| Option | Description | Selected |

| Remove only | No rename, use git repo name | ✓ |

**User's choice:** Remove only. Removing a project closes all its sessions.
