# Phase 04.1 -- UI Design Contract

> Visual and interaction contract for the atom state foundation + sidebar shell phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

- **Tool:** shadcn v4.1.0
- **Preset:** base-maia (style: base-maia, base color: neutral)
- **Component library:** base-ui (via @base-ui/react)
- **Icon library:** hugeicons (@hugeicons/react + @hugeicons/core-free-icons)
- **Font:** System font stack (Tailwind default -- no custom font loaded)

---

## Spacing Scale

Declared values (must be multiples of 4):

- **xs (4px):** Icon gaps, inline padding, status indicator margins
- **sm (8px):** Sidebar menu item gap (`gap-1` = 4px between items, `p-2` = 8px group padding), compact element spacing
- **md (16px):** Chat message area padding (`p-4`), form padding, default element spacing
- **lg (24px):** Not actively used in this phase
- **xl (32px):** Not actively used in this phase
- **2xl (48px):** Not actively used in this phase
- **3xl (64px):** Empty state vertical padding (`py-16`)

Exceptions: Sidebar width is `16rem` (256px) desktop / `18rem` (288px) mobile -- set by shadcn sidebar constants, not the spacing scale. The sidebar icon-collapsed width is `3rem` (48px).

---

## Typography

- **Body (14px / text-sm):** Weight 400 (regular), line-height 1.43 (20px). Used for chat messages, sidebar session labels, form input text.
- **Label (12px / text-xs):** Weight 500 (medium), line-height 1.33 (16px). Used for sidebar group labels (project headers), status text, error messages, sidebar menu badges.
- **Heading (16px / text-base):** Weight 600 (semibold), line-height 1.5 (24px). Used for sidebar header (app name), empty state heading.
- **Display:** Not used in this phase.

Note: The existing chat UI uses `text-sm` (14px) throughout. Sidebar inherits this via shadcn's built-in `text-sm` on `SidebarMenuButton` and `text-xs` on `SidebarGroupLabel`.

---

## Color

All colors are defined as CSS custom properties in `src/index.css` using OKLCH. Both light and dark themes are fully specified by the shadcn preset. This contract references the semantic token names, not raw values.

- **Dominant (60%) -- `var(--background)`:** Main content area background, chat panel surface
- **Secondary (30%) -- `var(--sidebar)` / `var(--muted)`:** Sidebar background, assistant message bubbles, hover states on sidebar items
- **Accent (10%) -- `var(--sidebar-accent)` / `var(--primary)`:** Reserved for: active session highlight in sidebar (`data-active` state on `SidebarMenuButton`), user message bubbles (`bg-primary`), send button, streaming cursor pulse
- **Destructive -- `var(--destructive)`:** Error indicator on session entries, error message banner in chat area

### Status indicator colors

- **Streaming:** `var(--sidebar-primary)` with CSS `animate-pulse`. A small filled circle (6px diameter) that pulses to indicate active streaming.
- **Unread:** `var(--sidebar-primary)` solid. A small filled circle (6px diameter), no animation.
- **Error:** `var(--destructive)` solid. A small filled circle (6px diameter), no animation.
- **Idle:** No indicator shown.

## Layout Contract

### Overall structure

```
SidebarProvider
  Sidebar (side="left", collapsible="offcanvas")
    SidebarHeader
      App name + SidebarTrigger
    SidebarContent (scrollable)
      For each project:
        Collapsible (defaultOpen)
          SidebarGroup
            SidebarGroupLabel (project name, acts as CollapsibleTrigger)
            CollapsibleContent
              SidebarGroupContent
                SidebarMenu
                  For each session:
                    SidebarMenuItem
                      SidebarMenuButton (isActive when selected)
                        Branch label (truncated)
                        SessionStatusIndicator
  SidebarInset
    Chat header bar (debug toggle, sidebar trigger on mobile)
    Chat message area (scrollable, flex-1)
    Error banner (conditional)
    Message input form (sticky bottom)
```

### Sidebar behavior

- **Desktop:** Sidebar is visible by default. Toggled via `Ctrl+B` keyboard shortcut or `SidebarTrigger` button. Collapses off-canvas.
- **Mobile:** Sidebar is hidden by default. Opens as a sheet overlay via `SidebarTrigger`.
- **Collapsible project groups:** Each project is wrapped in a `Collapsible` component, defaulting to open. Clicking the `SidebarGroupLabel` toggles the group. Chevron icon rotates to indicate state (down = open, right = collapsed).
- **Active session:** The currently selected session has `isActive={true}` on its `SidebarMenuButton`, which applies `bg-sidebar-accent` + `font-medium` styling.

### Chat panel behavior

- Chat panel fills remaining width via `SidebarInset` (flex-1).
- Message area scrolls vertically with `overflow-y-auto`.
- Auto-scroll to bottom on new messages: instant during streaming, smooth after completion.
- Input is disabled during streaming with placeholder "Waiting for response..."

## Interaction States

### Session entry in sidebar

- **Default:** `text-sidebar-foreground`, no background
- **Hover:** `bg-sidebar-accent text-sidebar-accent-foreground`
- **Active (selected):** `bg-sidebar-accent text-sidebar-accent-foreground font-medium` (via `data-active`)
- **With status indicator:** Status dot appears right-aligned within the menu button

### Project group header

- **Default:** `text-sidebar-foreground/70 text-xs font-medium` (shadcn `SidebarGroupLabel` default)
- **Hover:** No distinct hover -- group label acts as collapse trigger
- **Collapsed indicator:** Chevron icon rotates from pointing-down (open) to pointing-right (collapsed)

### Chat message input

- **Default:** Standard input styling, autofocus on mount
- **Disabled (streaming):** `disabled` attribute, placeholder changes, opacity reduced via disabled styles
- **Submit:** Enter key or click send button
