## Cross-Phase Integration

### Connected (17/18 chains verified)

The integration checker confirmed all major chains are wired:

1. **Database → EventStore → PersistentClaudeCli → SessionReconstructor** — Full chain from SQLite layer through event persistence to session reconstruction. Layer ordering correct in BaseLayer.
2. **RPC Layer — Four groups merged symmetrically** — ClaudeRpcGroup, PersistenceRpcGroup, GitRpcGroup, DialogRpcGroup merged in same order on server and client. All handler layers present in BaseLayer.
3. **Atom State — sendMessageAtom → CLI spawn → persistence → hydration** — Complete flow from user input through CLI process spawn, event persistence via Stream.tap, session ID persistence via updateTab RPC, and session hydration on app restart via reconstructSession.
4. **Tab Lifecycle — create → activate → stream → switch → restore** — All paths verified including cwd population, scroll position preservation, draft input persistence, and lazy hydration.
5. **Graceful Shutdown** — before-quit handler calls disposeAll() then runtime.dispose(). Ordering correct. In-flight stream termination works via main runtime fiber interruption.

### Orphaned Export (1)

**TabRuntimeManager.getOrCreate** — Defined and tested but never called from production paths. `disposeAll()` IS called from main.ts shutdown handler, but the HashMap is always empty. This is a documented scaffold for future per-tab CLI isolation.
